## 1. Что такое Git

Git — это распределённая система контроля версий, которая позволяет отслеживать изменения в коде и файлах проекта.  
С помощью Git можно работать с историей изменений, создавать ветки для отдельных задач и безопасно объединять изменения.  
Git упрощает командную работу и позволяет быстро откатиться к предыдущему состоянию проекта.  
В информационной безопасности Git важен для аудита изменений, контроля доступа и предотвращения утечек чувствительных данных.

## 5. Инцидент с secrets.txt

В рамках лабораторной работы был смоделирован инцидент утечки секрета — файл `secrets.txt` был случайно закоммичен в репозиторий.

### Обнаружение утечки
Утечка была обнаружена с помощью команд:
- `git log -- secrets.txt`
- `git show HEAD:secrets.txt`

### Исправление
Файл `secrets.txt` был добавлен в `.gitignore`, после чего удалён из репозитория командой `git rm` и закоммичен.

### Почему простого удаления недостаточно
Удаление файла из текущей версии репозитория не удаляет секрет из истории Git.  
Секрет остаётся доступным в предыдущих коммитах.

### Что требуется в реальной жизни
В реальном проекте необходимо:
- выполнить ротацию скомпрометированного секрета;
- переписать историю репозитория с помощью инструментов (git filter-repo или BFG);
- убедиться, что секрет больше нигде не используется.
